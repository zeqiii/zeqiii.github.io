Functions, Reserved Words and Macros

reinterpret_cast (expression)

Cast the expression to a new_type.

For example:

typedef int (*FunctionPointer)(int); 
int value = 21; 
FunctionPointer funcP; 
funcP = reinterpret_cast<FunctionPointer> (&value); 
funcP(value);

Of course, this program will crash after being executed.

Macro with variable argument list

For example:

#define LOG(format, ...) fprintf(stdout, format, __VA_ARGS__)

... means the arguments are variable, __VA_ARGS__ represents the variable arguments.

In GCC, we can also:

#define LOG(format, args...) fprintf(stdout, format, args)

With the macros defined above, we can:

LOG("%d,%s", 1, "hello")

size_t

Alias of one of the fundamental unsigned integer types. size_t is unsigned integer defined in standard C lib, in 64-bit OS, it is long unsigned integer.

sizeof operation returns a size_t type value.

size_t usually is used in array index, looping, pointer, memory location calculating.

mutable

mutable variables always can be changed even if it is in a const function.

As we know, if a function doesn’t change any fields of a class, this function will be declared with const. But sometimes this kind of functions need to modify some specific fields that have no business with the state of the class. In this condition, these specific fields should declared with the key word mutable.

volatile

To optimize a program, commonly, the compiler make the executables read/write a variable from registers. But sometimes a varible may be changed frequently.

If a variable are declared with the key word volatile, that means the variable can be changed at any moment, the compiler will compile the program to make this variable be read/write directly from the memory, which helps to avoid race condition.

misc tips
