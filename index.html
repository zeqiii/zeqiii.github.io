Analysis of cve-2015-3837

7 Mar, 2016

Introduction CVE-2015-3837 is another java deserialzation vulnerability in android. Wooyun had analyzed it. Description on CVE The OpenSSLX509Certificate class in org/conscrypt/OpenSSLX509Certificate.java in Android before 5.1.1 LMY48I improperly includes certain context data during serialization and deserialization, which allows attackers to execute arbitrary code via an application that sends a crafted Intent, aka internal bug 21437603. We know that Google had patched the bug disclosed in CVE-2014-7911, in which ObjectInputStream didn’t validate an object can be serialzed or not.

Analysis of cve-2015-6420

Apache Commons Collections Apache Commons Collections implements TransformedMap whose decorate() method can perform some operations on a Map. public static Map decorate(Map map, Transformer keyTransformer, Transformer valueTransformer) { return new TransformedMap(map, keyTransformer, valueTransformer); } Transformer is an implementation to transform an object. public interface Transformer { public Object transform(Object input); } Map has key and value. Parameters of decorate(), keyTransformer is responsible for the transformation of the keys of a Map, valueTransformer is responsible for values of a Map.

Analysis of cve-2014-7911

29 Feb, 2016

A brief look System_server is a process has system permission, to which one can send any serializable data. Although system_server won’t invoke any functions of the received objected, but finalize will be called when GC. Function finalize in android.os.BinderProxy calls a native function which has pointers, the pointers may be controlled by attackers. So GC may result in arbitrary code execution. android.os.BinderProxy isn’t serializable, however, java.io.ObjectInputStream doesn’t check whether an object really is serializable, attackers can craft a fake BinderProxy to acheive the goals.
